# Incident Response Checklist

## Purpose
This checklist provides a structured approach for responding to
security incidents in an enterprise IT environment, based on
Security+ principles.

---

## 1. Identification
- Detect and confirm the incident
- Identify affected systems, users, and data
- Collect initial evidence (logs, alerts, screenshots)

## 2. Containment
- Isolate affected systems from the network
- Disable compromised accounts
- Prevent further spread of the threat

## 3. Eradication
- Remove malware or unauthorized access
- Apply security patches and updates
- Reset passwords and credentials if required

## 4. Recovery
- Restore systems from clean backups
- Monitor systems for abnormal behavior
- Validate system functionality

## 5. Lessons Learned
- Document the incident and actions taken
- Identify gaps in controls or processes
- Recommend preventive measures

---

## Tools Referenced
- SIEM / logging tools
- Antivirus / endpoint protection
- Active Directory
- Firewall logs
